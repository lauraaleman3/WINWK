Resource Object:

Represent resources.

Requires at least the following top-level members:
1) id
2) type

Exception:

The id member is not required when the resource object originates at the client and represents a new resource to be created on the server.

Example for when id is not needed: User creates own id when making an account on a social media platform.

Resource Object may contain:

Other top-level members:
1) attributes
2) relationships
3) links
4) metas

Identification:

Requirements to identify resources:
1) id and type values have to be strings
2) Each object's type and id pair has to identify a single unique resource
3) Values of type members have to adhere to the "member names" rules
